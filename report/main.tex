\begin{abstract}
We report the final implemented audio generation system for visual speech
inputs. The delivered system converts talking-face video into speech by
combining an Auto-AVSR visual encoder, SmolLM2 text hidden states, speaker
identity features, and an explicit timbre/audio prompt derived from Mimi audio
latents. Although the original exploration included flow-matching sampling, the
best working route is a deterministic reconstruction model: a frozen base
reconstructor predicts normalized Mimi latents and a residual head corrects the
base prediction. The final checkpoint reaches a full validation latent
correlation of 0.5818 on LRS3 with ground-truth text timestamps, and replacing
ground-truth text with Auto-AVSR text only reduces correlation by about 0.0035.
We also provide a raw-video inference pipeline, a Gradio interface, and a silent
video mode that accepts optional reference audio for timbre control while
preserving the full input video duration.
\end{abstract}

%======================================================================
\section{Introduction}
%======================================================================

The goal of this project is to synthesize intelligible speech from visual speech
evidence. In the final implemented system, the input can be either a talking
video with audio or a silent video with optional reference audio. The output is a
waveform decoded by Mimi and, for demonstrations, an MP4 with the generated
audio muxed back onto the face video.

The project began with a broader streaming lip-to-speech objective. The final
deliverable is narrower and more empirical: a robust offline AVSR-conditioned
audio reconstruction pipeline with explicit timbre control. This report
therefore focuses only on implemented components, measured behavior, and the
current engineering interface.

The final system makes four practical contributions:
\begin{enumerate}
    \item \textbf{AVSR-compatible visual preprocessing.} We replaced the earlier
    RGB lip input with the correct Auto-AVSR-style grayscale mouth crop
    \texttt{lip\_avsr.npy}, improving raw-video behavior on external examples.
    \item \textbf{Deterministic residual audio-latent reconstruction.} We found
    that the strongest route is not stochastic flow sampling but endpoint
    regression in normalized Mimi latent space with a frozen base and residual
    correction.
    \item \textbf{Explicit timbre/audio prompt conditioning.} The model uses the
    first 38 Mimi latent frames, about 3.04 seconds, both as prompt tokens and as
    global mean/std timbre statistics. A new silent-video mode can instead use a
    separate reference audio file for these conditions.
    \item \textbf{One-command demo pipeline.} A single script now performs video
    standardization, face/lip preprocessing, AVSR feature extraction, SmolLM2
    hidden extraction, timbre conditioning, FM-AVSR reconstruction, Mimi decode,
    and MP4 muxing.
\end{enumerate}

%======================================================================
\section{Goal of Achievement}
%======================================================================

The achieved goal is a working visual-speech-to-audio reconstruction system with
documented model checkpoints, quantitative evaluation, raw-video examples, and a
usable command-line/GUI interface. The system is not claimed to be real-time and
is not a production voice-cloning model; instead, it is a validated research
prototype for generating Mimi audio latents from visual, text, speaker, and
timbre conditions.

The final acceptance criteria were:
\begin{itemize}
    \item generate audible speech from LRS3 validation clips and external raw
    videos;
    \item remove the first 3.04 seconds from listening examples when the same
    clip is used as timbre/audio prompt;
    \item support Auto-AVSR text as a replacement for ground-truth transcript at
    inference time;
    \item support a silent-video mode with optional reference audio while keeping
    the full video length;
    \item keep the codebase command surface focused on the active FM-AVSR route.
\end{itemize}

\section{Methods}
%======================================================================

\subsection{System Overview}

Figure~\ref{fig:architecture} summarizes the implemented pipeline. The video
path first produces aligned face crops and an AVSR-compatible mouth crop. The
mouth crop is encoded by Auto-AVSR into visual features and a CTC-decoded text
string. The text is passed through SmolLM2 to obtain hidden states. Speaker
identity is extracted from aligned face crops. Audio conditioning is derived
from either the input audio prefix or an external reference audio file.

\begin{figure}[h]
    \centering
    \includegraphics[width=\linewidth]{assets/system_architecture.png}
    \caption{Implemented FM-AVSR audio reconstruction pipeline. Visual, text,
    speaker, timbre, and audio-prompt conditions are fused by the reconstruction
    model to predict normalized Mimi latents, which are decoded to waveform and
    muxed back to video.}
    \label{fig:architecture}
\end{figure}

\subsection{Data and Latent Representation}

The final route uses the following per-clip files:
\begin{table}[h]
\centering
\small
\begin{tabular}{@{}lll@{}}
\toprule
\textbf{File} & \textbf{Shape} & \textbf{Role} \\
\midrule
\texttt{lip\_avsr.npy} & $(T_v,96,96)$ & Auto-AVSR-compatible grayscale mouth crop \\
\texttt{avsr\_enc\_lipavsr.npy} & $(T_v,768)$ & visual condition \\
\texttt{avsr\_text\_lipavsr.txt} & text & Auto-AVSR transcript for raw video \\
\texttt{smollm2\_h\_lipavsr.npy} & $(L,960)$ & SmolLM2 hidden states for Auto-AVSR text \\
\texttt{speaker\_emb.npy} & $(256)$ & face-based speaker/identity condition \\
\texttt{latent.npz} & $(T_a,512)$ & Mimi continuous latent target or dummy target \\
\texttt{audio\_prompt.npy} & $(38,512)$ & normalized Mimi prompt tokens \\
\texttt{timbre\_cond.npy} & $(1024)$ & prompt mean and standard deviation \\
\bottomrule
\end{tabular}
\caption{Core files consumed by the final reconstruction path.}
\end{table}

Mimi latents are normalized per dimension:
\begin{equation}
    y_t = \frac{z_t - \mu}{\sigma}, \qquad z_t \in \mathbb{R}^{512},
\end{equation}
where $z_t$ is the continuous Mimi latent before decoding. Metrics are computed
in normalized latent space. Generated latents are denormalized and decoded by
the Mimi decoder:
\begin{equation}
    \hat{z}_t = \hat{y}_t \sigma + \mu, \qquad
    \hat{\mathbf{x}}_{\mathrm{wav}} = \mathrm{MimiDecoder}(\hat{z}_{1:T}).
\end{equation}

\subsection{Condition Construction}

Auto-AVSR features are produced at approximately 25 Hz, while Mimi latents are
used at 12.5 Hz. We align by taking every second visual frame:
\begin{equation}
    v_t = \mathrm{AVSR}(V)_{2t}, \qquad v_t \in \mathbb{R}^{768}.
\end{equation}

Text hidden states are obtained by encoding either ground-truth transcript
(\texttt{text\_json}) or Auto-AVSR transcript (\texttt{lipavsr}) with SmolLM2.
For in-distribution LRS3 evaluation, word timestamps map audio-latent frames to
language-model positions. For external raw videos, no ground-truth timestamps
exist, so hidden states are uniformly resampled.

The timbre condition is computed from normalized prompt latents $A \in
\mathbb{R}^{38 \times 512}$:
\begin{equation}
    q = [\mathrm{mean}(A), \mathrm{std}(A)] \in \mathbb{R}^{1024}.
\end{equation}
The prompt tokens $A$ are also provided directly as cross-attention condition
tokens. In silent-video mode, $A$ and $q$ are derived from optional reference
audio; if no reference is provided, both are zero.

\subsection{Reconstruction Model}

The strongest model is a deterministic endpoint reconstructor rather than a
sampling model. Figure~\ref{fig:recon} contrasts the original flow-matching
sampling route with the implemented route. The current path calls
\texttt{reconstruct\_from\_cond()} with
\begin{equation}
    \tilde{x} = 0,\qquad \tau = 1.
\end{equation}
Thus the network predicts an endpoint directly from condition $C$:
\begin{equation}
    y_{\mathrm{base}} = f_{\mathrm{base}}(C), \qquad
    \Delta = f_{\mathrm{residual}}(C), \qquad
    \hat{y} = y_{\mathrm{base}} + \Delta.
\end{equation}
There is no random noise input at inference time and no iterative denoising
loop. The residual base is a frozen recon checkpoint, and the final checkpoint
learns the correction term.

\begin{figure}[h]
    \centering
    \includegraphics[width=\linewidth]{assets/recon_vs_sampling.png}
    \caption{The final system uses deterministic residual reconstruction. The
    original FM sampling route learns a vector field from noise to target; the
    final route fixes $\tilde{x}=0$ and $\tau=1$, then predicts an endpoint with
    a frozen base plus residual correction.}
    \label{fig:recon}
\end{figure}

\subsection{Silent Video with Reference Audio}

The latest interface supports a silent input video and optional reference audio.
The video is processed without requiring an original audio track. A temporary
silent audio track is added only so the legacy face/audio preprocessing worker
can run. The target latent is a zero dummy latent with length determined by the
video duration; it is used only to set output length and metrics bookkeeping.
When a reference audio file is provided, it is converted to 24 kHz mono WAV,
encoded by Mimi, normalized, and used to populate \texttt{audio\_prompt.npy} and
\texttt{timbre\_cond.npy}. Output muxing does not crop the first 3.04 seconds,
so the generated MP4 preserves the full input duration.

\begin{figure}[h]
    \centering
    \includegraphics[width=\linewidth]{assets/silent_ref_pipeline.png}
    \caption{Silent-video inference mode. Optional reference audio controls
    timbre and prompt conditions. If the reference is absent, zero prompt and
    zero timbre are used. The output video keeps the full input duration and is
    saved as \texttt{\string<exp\string>\_pred\_full.mp4}.}
    \label{fig:silent_ref}
\end{figure}

\section{Engineering Deliverables}
%======================================================================

\paragraph{Command-line pipeline.} The primary entry point is:
\begin{verbatim}
python scripts/run_raw_video_avsr_recon_pipeline.py \
  --input data/trump.mov --exp trump_cleanup_verify --force
\end{verbatim}
For silent video with reference audio:
\begin{verbatim}
python scripts/run_raw_video_avsr_recon_pipeline.py \
  --input silent.mp4 --ref_audio ref.wav \
  --silent_input --exp demo --force
\end{verbatim}

\paragraph{GUI.} \texttt{scripts/gradio\_avsr\_gui.py} exposes video upload,
optional reference audio, and silent-input controls. It calls the same backend
pipeline.

\paragraph{Codebase cleanup.} The active \texttt{scripts/} surface now contains
only the FM-AVSR route and raw-video utilities. Historical v2/v3/v4,
Mimi-code, teacher-cache, and sweep scripts were moved into
\texttt{archive/scripts/}.

\paragraph{Tests.} The final branch includes focused tests for dataset text
sources, evaluation helpers, timbre/audio prompt handling, and raw-video
pipeline helper behavior. The final verification command ran 97 unit tests
successfully.

\section{Limitations}
%======================================================================

\begin{itemize}
    \item The system is offline and not real-time. Previous timing showed that
    face alignment and lip cropping dominate runtime; the FM head and Mimi
    decoder are not the bottleneck.
    \item The final model is deterministic. It improves stability but can
    average speaker-specific acoustic detail, which partly explains the
    ``mixed voice'' artifacts observed in listening tests.
    \item External videos remain harder than LRS3 validation clips because of
    domain shift, missing word timestamps, and crop quality differences.
    \item The silent-video reference-audio mode controls timbre through latent
    prompt statistics; it is not a full speaker cloning model and does not
    guarantee high-fidelity identity preservation.
    \item Metrics in silent mode compare against a dummy zero latent and are
    only bookkeeping, not a quality estimate.
\end{itemize}

%======================================================================
\section{Conclusion}
%======================================================================

The final delivered system is a practical AVSR-conditioned audio reconstruction
pipeline. Its main lesson is that, for the available data and model capacity,
deterministic Mimi-latent reconstruction with explicit timbre/audio prompt
conditioning is stronger and more reliable than the explored stochastic FM
sampling route. Correct Auto-AVSR-compatible preprocessing is essential, and
text-source quality has a smaller measured effect than visual and timbre
conditioning in the final architecture. The resulting codebase now supports
both standard talking-video demos and silent-video demos with optional reference
audio, making the system suitable for final presentation and further targeted
improvement.
